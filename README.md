## 🌳 Flujo de Trabajo (GitFlow)

Este proyecto utiliza una estructura de ramas organizada para mantener la estabilidad del código:

*   **`main`**: Código en producción, siempre estable.
*   **`dev/`**: Rama principal de integración. Aquí se fusionan todas las funcionalidades terminadas antes de ir a producción.
*   **`front/`**: Desarrollo de nuevas interfaces, componentes y estilos.
*   **`docker/`**: Configuraciones de infraestructura, archivos `Dockerfile` y `docker-compose`.
*   **`fix/`**: Correcciones de errores (bugs) encontrados en `dev` o `main`.
*   **`doc/`**: Actualizaciones de documentación, guías de usuario y cambios en este README.
